# omc-routing

어떤 작업을 어느 에이전트/서브에이전트로 라우팅할지에 대한 규칙 문서. 에이전트 역할은 [AGENTS.md](AGENTS.md) 참고.

## 기본 라우팅 규칙 (하네스 기본값)

- **단일 파일/알고 있는 대상**: Read/Grep/Glob을 직접 사용, 에이전트로 위임하지 않음.
- **3회 쿼리 이상 필요한 폭넓은 코드 탐색·리서치**: `Explore` 서브에이전트.
- **여러 단계로 이뤄진 복잡한 작업, 결과가 커서 메인 컨텍스트를 오염시킬 수 있는 조사**: `general-purpose` 에이전트.
- **구현 전략 설계/아키텍처 트레이드오프 검토**: `Plan` 에이전트.
- **Claude Code/Agent SDK/Claude API/Claude Tag 관련 질문**: `claude-code-guide`.
- **명시적으로 이름이 불린 서브에이전트**: 사용자가 지정한 subagent_type을 그대로 사용.
- **병렬화 가능한 독립 작업**: 한 메시지에 여러 Agent 호출을 동시에 담아 병렬 실행.
- **단순 질문/명확한 한 가지 작업**: 에이전트 스폰 없이 메인 스레드에서 직접 처리.
