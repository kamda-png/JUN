# omc-routing

어떤 작업을 어느 에이전트/서브에이전트로 라우팅할지에 대한 규칙 문서. 에이전트 각각의 역할은 [AGENTS.md](AGENTS.md) 참고. 하네스에 이미 문서화된 기본 규칙 위주로 시작하고, 실제로 이 프로젝트에서 적용해본 뒤 검증된 규칙만 추가한다.

## 기본 라우팅 규칙 (하네스 기본값)

- **단일 파일/알고 있는 대상**: 경로를 이미 알면 Explore나 다른 에이전트를 거치지 않고 Read/Grep/Glob을 직접 사용한다.
- **3회 쿼리 이상이 필요한 폭넓은 코드 탐색·리서치**: `Explore` 서브에이전트로 위임 ("이 심볼이 어디서 쓰이나", "이런 이름 규칙의 파일이 어디 있나" 류). 탐색 폭에 따라 quick/medium/very thorough를 지정.
- **여러 단계로 이뤄진 복잡한 작업, 또는 결과가 크고 메인 컨텍스트를 오염시킬 수 있는 조사**: `general-purpose` 에이전트.
- **구현 전략 설계/아키텍처 트레이드오프 검토**: `Plan` 에이전트 (planning 단계에서만, 실제 코드 작성 전).
- **Claude Code/Agent SDK/Claude API/Claude Tag 관련 질문**: `claude-code-guide`. 답을 기억에서 바로 지어내지 말고 위임.
- **명시적으로 이름이 불린 서브에이전트**: 사용자가 특정 subagent_type을 지정하면 그대로 사용.
- **병렬화 가능한 독립 작업**: 서로 의존성 없는 여러 조사는 한 메시지에 여러 Agent 호출을 동시에 담아 병렬 실행.
- **단순 질문/명확한 한 가지 작업**: 에이전트로 위임하지 않고 메인 스레드에서 직접 처리 (에이전트 스폰은 콜드 스타트 비용이 있으므로 남용하지 않음).
